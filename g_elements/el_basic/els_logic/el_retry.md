# Повтор попыток

*Eng: Retry*

![](<../../../.gitbook/assets/image (161).png>)

Элемент представляет собой цикл, который выполняет попытки выполнения действия либо заданное число раз, либо до тех пор, пока выполняется определенное условие. Это важное средство в автоматизации, позволяющее обеспечить надежное и управляемое выполнение действий, даже при возникновении ошибок или непредвиденных ситуаций

### Свойства:

Символ `*` в названии свойства указывает на обязательность заполнения. Описание общих свойств см. в разделе [Свойства элемента](https://docs.primo-rpa.ru/primo-rpa/primo-studio/process/elements#svoistva-elementa).

| Свойство  | Тип     | Описание                      |
| --------- | ------- | ----------------------------- |
| Условие\* | Boolean | Условие выхода из цикла . Если данное условие истинно, активность завершит выполнение     |
| Число     | Int32   | Число попыток. Свойство "Число" определяет количество попыток выполнения задачи или действия. Если задано число попыток и оно не равно нулю, активность "Повтор" будет выполнять задачу указанное количество раз                 |
| Интервал  | Int32   | Интервал между попытками. Свойство определяет временной интервал (в миллисекундах) между попытками выполнения задачи. Полезен, например, для снижения нагрузки на систему или учета времени ожидания.|


###  Learning

Для изучения работы с элементом **Повтор попыток**, вы можете скачать обучающий RPA-проект по следующей ссылке: [Скачать архив Learning](https://github.com/PrimoRPA/Learning/archive/refs/heads/master.zip)

1. Скачайте архив с обучающими материалами с указанной страницы.
2. Распакуйте архив и откройте проект `StudioActivities` в вашей среде разработки.
3. Найдите процесс `StudioActivities/Ru/Управление/Циклы.ltw` для изучения работы элемента.

В релизе 23.11 был улучшен функционал свойства "Продолжить при ошибке" для элемента **Повтор попыток (Retry)**, позволяющий элементу продолжать свои действия при возникновении исключения (Exception). 
Если данное свойство установлено в значение **True**, выполнение процесса будет продолжаться, несмотря на возможные ошибки внутри активности. 

**Обратите внимание**, что галка "Приостанавливать отладку на исключении" в настройках Студии не будет учитываться, если элемент "Повтор попыток" настроен на "Продолжить при ошибке".
