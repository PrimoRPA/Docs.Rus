# Добавление RPA-проектов

Добавить RPA-проект в Оркестратор можно двумя способами: [через Студию](https://docs.primo-rpa.ru/primo-rpa/primo-studio/projects/publish) и непосредственно в Оркестраторе. Рекомендуется опубликовывать проект через Студию за исключением случаев, когда к ней нет доступа либо когда требуется что-то удалить из пакета проекта. В таком случае можно удалить ненужные файлы и загрузить архив в Оркестраторе.

В Оркестраторе добавление проекта осуществляется на вкладке **RPA-проекты ➝ Все RPA-проекты** при нажатии на кнопку **Добавить RPA-проект**:

![](<../../.gitbook/assets/Добавить проект. Орк.png>)

В открывшейся форме добавьте архив проекта и укажите необходимые параметры:
* **Наименование** - название проекта должно состоять только из букв, цифр, дефиса, символа подчеркивания и точки. Без использования пробелов.
* **Приоритет** - определяет, как долго проект будет находиться в очереди на выполнение, а также в каком порядке проекты будут выходить из нее при одновременном помещении. Значение 0 - это высший приоритет. Время нахождения проекта в очереди (в сек.) указывается в конфигурационном файле:
* **Использовать заглушки** - в Студии есть элемент [Заглушка](https://docs.primo-rpa.ru/primo-rpa/g_elements/osnovnye-elementy/testing/mock), там же в дебаге есть заглушки(призрак). если включен и включена заглушка, то по пути на элементе будет правый путь. если выключен - левый. допустим, я не хоч записывать что-то в базу данных и просто тестирую. 
* Конфигурация запуска: 
    * **Стандартная** - выберите одно из значений: Отсутствует (по умолчанию), Отладка или Релиз.
    * **Специальная** - ? введите значение
    * **Версии дистрибутива робота** - если RPA-проект может выполняться только Роботами определенных версий, необходимо привязать эти версии дистрибутива к RPA-проекту
    * **Закрыть RPD-сессию** - если настройка включена, то после выполнения проекта Робот закроет сессию. По умолчанию неактивна - рекомендуется использовать настройку осторожно, поскольку закрытие сессии может негативно повлиять на работу других Роботов в той же сессии.
    * **Запуск в единственном экземпляре** - ?
    * **Не повторять в очереди ожидания** - если параметр включен, проект может быть помещен в очередь только один раз. 

Загруженный проект отобразится в общем списке. К нему можно добавить версию - для этого выделите существующий проект и используйте кнопку **Добавить версию** проекта:
