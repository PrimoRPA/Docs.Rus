# Базовые понятия

В данном разделе вы познакомитесь с базовыми понятиями, используемыми при работе с веб-приложением Primo AI. 

Ключевыми понятиями в машинном обучении являются:
* **Обучение** — процесс тренировки модели на обучающей выборке для оптимизации её параметров,  
* **инференс** — использование обученной модели для предсказания результатов на новых данных.

В нашем контексте 'данные' - это изображения, сканы документов.

## Обучение

| Термин                    | Определение                                                                                                                                                                                                                                                                             |
|---------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Augmentation (Аугментация)| Техника увеличения разнообразия обучающих данных путем применения различных преобразований к исходным изображениям, таких как вращение, изменение яркости, добавление шума и другие. Аугментация помогает улучшить обобщающую способность модели, предотвращая переобучение.             |
| Базовая модель| Заранее предобученная модель, готовая извлекать определенные свойства (features) из изображений.|
| Transfer Learning (Перенос обучения) | Метод машинного обучения, при котором модель, предобученная на одной задаче, используется в качестве отправной точки для другой, связанной задачи. Этот подход позволяет использовать уже обученные веса и параметры **базовой модели**, что может значительно сократить время и ресурсы, необходимые для обучения на новой задаче (детекция полей в документе), особенно когда доступно ограниченное количество данных. |
| Batch Size (Размер пакета)| Количество изображений, проходящих через модель одновременно при единовременном обновлении градиента. Увеличение размера пакета может ускорить обучение, но требует больше памяти. Если ООМ (out of memory), то, первым делом, уменьшаем batch_size.                                                                                                                    |
| Dataset (Датасет)         | Набор изображений, используемый для обучения и тестирования модели. В контексте обнаружения объектов датасет обычно состоит из изображений и аннотаций, описывающих местоположение и классы объектов (поля в документах)  на этих изображениях.                                                     |
| Epoch (Эпоха)             | Один полный проход через весь набор обучающих картинок. Во время одной эпохи модель видит каждую картинку в обучающем наборе данных один раз.                                                                                                                                                |
| Learning Rate (LR)        | Скорость обучения - параметр, определяющий размер шага, с которым обновляются параметры модели во время обучения. Learning rate влияет на скорость сходимости и качество итоговой модели.                                                                                               |
| Max_iterations            | Максимальное количество итераций, за которое модель обучается. Одна итерация включает один шаг обновления параметров модели на основе одной или нескольких партий данных (batch_size).                                                                                                              |
| AP (Average precision)   | Средняя точность - это метрика, используемая для оценки качества модели обнаружения объектов. AP рассчитывается как среднее значение точности для различных уровней полноты (recall). |
| Model Weights (Веса модели)| Параметры, которые модель рассчитывает в процессе обучения. Веса модели сохраняются и используются при инференсе для предсказания на новых данных.                                                                                                                                             |
| Num_workers (Количество потоков)| Количество потоков, используемых для загрузки данных. Увеличение числа потоков может ускорить обучение, особенно при использовании больших датасетов.                                                                                                                               |
| Overfitting (Переобучение)| Ситуация, когда модель слишком хорошо запоминает обучающие данные, но плохо обобщает новые данные. Переобученная модель имеет высокую точность на обучающем наборе, но низкую точность на тестовом наборе.                                                                       |
| PyTorch                   | Популярный фреймворк для разработки и обучения нейронных сетей, разработанный. PyTorch предоставляет удобные инструменты для построения и обучения моделей, а также гибкие возможности для отладки.                                                                                                                         |
| Training (Обучение)       | Процесс настройки параметров нейронной сети (например, weights and biases) с использованием обучающих данных. В ходе обучения сеть постепенно улучшает свои предсказания, минимизируя ошибки.                                                                                           |
| Тестирование   | Этап проверки качества модели на отдельном наборе данных, который не использовался для обучения. Тестирование помогает оценить обобщающую способность модели и выявить переобучение.   

## Инференс

| Термин                    | Определение                                                                                                                                                                                                                                                                             |
|---------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Bounding Box (Описывающий прямоугольник)| Прямоугольник, который минимально охватывает объект на изображении. Bounding box используется для обозначения местоположения и размера объекта.                                                                                                                                                                       |
| CNN (Сверточная нейронная сеть)| Тип нейронной сети, специально разработанный для обработки данных,  таких как изображения. CNN эффективно захватывает пространственные и временные зависимости в данных с помощью использования фильтров и сверточных слоев.                                                       |
| Generalization (Обобщение)| Способность модели хорошо предсказывать новые, ранее не известные данные. Хорошо обобщающая модель успешно справляется с задачей на различных наборах данных, а не только на тех, которые использовались для обучения.                                                                |
| Inference (Инференс)      | Процесс использования обученной модели для предсказания на новых данных. В контексте обнаружения объектов инференс включает определение классов объектов (поля в документах) и их местоположений на изображении.                                                                                           |
| Localization (Локализация)| Процесс определения точного местоположения объекта на изображении путем предсказания координат ограничивающего прямоугольника (bounding box), окружающего объект.                                                                                                                     |
| Neuron (Нейрон)           | Основной элемент нейронной сети, который принимает входные сигналы, выполняет вычисления и передает выходные сигналы дальше по сети. Нейроны организованы в слои и соединены между собой.                                                                                                |
| Object Detection (Обнаружение объектов)| Процесс идентификации и локализации объектов на изображении. Система не только определяет, что на изображении находится объект определенного класса (например, поле - Наименование поставщика), но и указывает его точное местоположение.                                       |
| Testing (Тестирование)    | Процесс оценки производительности обученной модели на отдельном наборе данных, который не использовался для обучения. Тестирование помогает определить, насколько хорошо модель может предсказывать новые данные.                                                         |
| Trained Model (Обученная модель)| Модель, которая прошла процесс обучения и теперь может использоваться для инференса (предсказания) на новых данных. Обученная модель имеет настроенные веса и параметры, оптимизированные в ходе тренировки.                                                                       |




Датасет - Обработанная и структурированная информация, представленная в табличном виде, называется Dataset. В такой таблице объектами называются строки, а признаками – столбцы. Совокупность этой информации называется размеченными данными, которые являются основой для машинного обучения.

Датасет - Совокупность данных, систематизированных в определённом формате. //Датасеты служат основой для обучения моделей машинного обучения. Они предоставляют информацию, на основе которой алгоритмы учатся находить закономерности, предсказывать результаты или классифицировать объекты. Важно отметить, что качество и релевантность датасета во многом определяют эффективность обучения модели.
//Датасет представляет собой набор данных, которые используются в различных видах анализа и машинного обучения. Причем успешность последнего напрямую зависит от объема исходной информации: чем ее больше, тем качественней будет развиваться ИИ.
