# Обзор интерфейса

В статье представлена информация об основных функциональных элементах в веб-интерфейсе Idea Hub. Статья не ставит перед собой цель полностью описать все возможные настройки и функции каждого раздела.


## Меню

При входе в Idea Hub открывается главная страница. В зависимости от уровня доступа к данным, установленного для конкретного пользователя, внешний вид страницы может различаться.

С левой стороны экрана расположены боковые панели меню:

![](<../../idea-hub/resources/getting-started/admin-menu.png>)

**Административное меню** (1) отображается только пользователю с ролью администратора. Меню предназначено для настройки продукта под нужды конкретной организации. Например, через него можно настроить конфигурацию, создать пользователей, назначить права доступа к данным, установить необходимые расширения. Поскольку это меню не всегда необходимо использовать, внизу находится кнопка, по которой его можно быстро свернуть и таким образом расширить рабочую область. 

**Пользовательское меню** (2) обеспечивает доступ пользователей к функциональным возможностям продукта. Более подробное описание каждого из пунктов меню приведено ниже.


### Главная

На главной странице по умолчанию отображается дашборд с информационными панелями, где собраны и отображены ключевые данные об автоматизации в организации. Они помогают оценить ситуацию в целом, без детализации по отдельным процессам или департаментам.

Кроме того, Idea Hub позволяет переопределить главную страницу — администратор может назначить любую страницу в качестве главной. Настройка устанавливается в административном меню: **Конфигурация > Система > Основные настройки сайта > поле Главная страница**. 

![](<../../idea-hub/resources/getting-started/main.png>)

Дашборд главной страницы отображает предзаданные панели от команды Primo RPA. Для каждой панели существуют инструменты для гибкой настройки — вы можете количество показателей, формулы расчета, удалить панель, переместить вверх-вниз или добавить новую панель к уже существующим.

**Цифрами на рисунке отмечены:**

<details>

<summary>1 — Панель с обобщенными данными</summary>

* `Всего Ч/М` — какое количество человеко-месяцев было сокращено роботами за весь период работы роботов.
* `Фактический эффект` — экономический эффект в рублях, который организация получила в результате замены человеко-месяцев автоматизацией. 
* `Всего успешных операций` — суммарное количество операций, которые роботы выполнили за все время эксплуатации. Операция в данном случае — это 1 экземпляр процесса, который выполнил робот без ошибок. Данные по успешным операциям поступают из Оркестратора.

Под каждым из этих показателей подсвечена разница эффектов текущего года относительно последнего дня прошлого года.

</details>

<details>

<summary>2 — Панель с экономическими показателями за год и общей информацией о процессах</summary>

**Первая карточка**:
* `Эффект в деньгах за текущий год` — фактическая экономия в рублях за текущий год, которую организация получила от сокращения человеко-месяцев. Рядом отображается дельта, которая показывает разницу относительно предыдущего года.
* `Эффект Ч/М за год` — фактическое количество человеко-месяцев за год, которое было сокращено при помощи автоматизации.
* `Утилизация лицензий за год` — отражает, насколько эффективно использовались лицензии роботов за год. Высокая утилизация может свидетельствовать о том, что лицензии используются эффективно, тогда как низкая может указывать на избыточные расходы. Данные по утилизации собираются из Оркестратора и выводятся в виде двух показателей:
   *  Коэффициент утилизации — количество дней в году, в течение которых использовались лицензии роботов. Например, у вас было 10 лицензий, длительность каждой из которых составляла 1 год. Значит 100% времени для 10 лицензий составляет 3650 дней. Если бы все лицензии использовались 24 часа в сутки, то данный показатель бы отразил значение `3650`.
   *  Процент — отношение фактического использования лицензий к общему количеству доступных лицензий. Например, если у вас есть 10 лицензий, и в течение года использовалось 8 из них, то утилизация лицензий составит 80%. 

**Вторая карточка**:
* `Всего процессов в эксплуатации` — сколько процессов на текущий момент находится в статусе эксплуатации. 
* `Всего автоматизируемых систем` — общее количество внешних систем, которые были задействованы в процессах. Например, это такие системы, как 1С, SAP и т.д.
* `Всего автоматизируемых отделов` — общее количество департаментов, в которых используются процессы в статусе эксплуатации. 

**Воронка автоматизации** визуально отражает, на каких стадиях находятся все имеющиеся процессы в организации:
* `Заявки` — количество процессов, которые находятся на стадии предложенных Идей.
* `В разработке` — количество процессов в разработке на текущий момент.
* `В проде` — количество процессов в эксплуатации на текущий момент.

</details>

* `Эффект от роботизации в месяц` — количество сокращенных трудозатрат за завершенный месяц. Ниже отображается сравнительный показатель по отношению к предыдыдущему месяцу.
* `Эффект от роботизации в квартал` — количество сокращенных трудозатрат за завершенный квартал. Ниже отображается сравнительный показатель по отношению к предыдыдущему кварталу.

</details>

<summary>4 — Элементы управления панелью</summary>

</details>




### Департаменты

При нажатии на пункт меню **Департаменты** открывается страница с перечнем департаментов организации. Департаменты могут быть добавлены в Idea Hub как с помощью импорта файла `areas.xlsx`, так и вручную через административное меню или Active Directory.

По умолчанию отображаются только те департаменты, с которыми связаны какие-либо процессы. Если включить параметр **Показать департаменты без процессов** также отобразятся департаменты, с которыми пока не связан ни один процесс.

![](<../../idea-hub/resources/departments.png>)

Для каждого департамента доступна детализированная информация о связанных процессах и рассчитанных экономических эффектах. Чтобы увидеть эту информацию, нажмите на название нужного департамента.

Например, на рисунке ниже отображены процессы для департамента **Безопасность и аудит**. Вы можете перейти в нужный процесс департамента прямо с этой страницы — просто выберите карточку процесса.

![](<../../idea-hub/resources/dep-inside.png>)



### Процессы

На странице **Процессы** собрана информация обо всех роботизированных процессах в организации и рассчитанных для них показателях. Процессы могут быть добавлены в Idea Hub с помощью импорта файла `process.xlsx` или вручную через предложение Идеи.

Ключевая особенность Idea Hub заключается в том, что вы можете собирать статистику процессов из Оркестратора и гибко управлять ее отображением за счет такого механизма как формулы.

Как это работает:
1. Статистика из Оркестратора собирается ежедневно, данные по каждому дню хранятся в базе данных.
2. На основе статистики ежедневно рассчитываются показатели для конкретного процесса. Например, для расчета фактического FTE берутся данные за каждый день.
3. С помощью формул можно определить, какие значения следует выводить в показателях процессов. Например, поскольку данные по фактическому FTE хранятся на протяжении всего срока эксплуатации, вы можете:
   * расчитать FTE за месяц и вывести этот показатель в качестве результата эффективности процесса;
   * расчитать FTE за любой другой срок — год, квартал, неделю  — и отобразить его в карточке процесса.
  
**Цифрами на рисунке ниже отмечены:**

![](<../../idea-hub/resources/getting-started/processes-page.png>)

<details>

<summary>1 — Кнопки, которые отвечают за представление процессов на странице </summary>

По умолчанию процессы отображаются в виде карточек, но вы можете переключиться на табличное представление. При выборе кнопки **В эксплуатации** отобразятся только процессы со статусом `Эксплуатация`.

</details>


<details>

<summary>2 — Строка для поиска процессов по текстовым полям</summary>

Для корректной работы поиска необходимо указывать слово полностью. 

</details>

<details>

<summary>3 — Карточки автоматизируемых процессов </summary>

Карточка процесса по умолчанию содержит следующие поля:
* `Департамент` — отдел, к которому относится процесс.
* `Системы` — внешние системы, которые задействованы в автоматизации процесса. Например, 1С, SAP и другие.
* `Статус` — текущий статус процесса. Статусы описывают жизненный цикл процесса внутри организации, начиная с этапа создания заявки на роботизацию (Идеация).
* `Оркестратор подключен` — параметр отображает, собирает ли процесс статистику из Оркестратора.

Под карточкой дополнительно отображаются показатели эффектов: 
* `Плановый FTE` — сокращаемый FTE, которое рассчитал владелец процесса, когда подавал идею на роботизацию. 1 FTE — эквивалент полной занятности сотрудника в месяц или, другими словами, количество человеческого труда, которое планируется сократить за счет автоматизации.
* `Фактический FTE` — усредненный FTE в месяц за все время эксплуатации процесса. В скобках отображается разница — насколько фактический FTE выше/ниже планового. Значение данного показателя рассчитывается по формуле. Администратор может изменить формулу, а также отображение показателя на свое усмотрение.
* `Утилизация лицензий` — усредненный показатель утилизации лицензий роботов в месяц, который рассчитывается за все время эксплуатации процесса. Значение зависит от статуса процесса: если процесс на стадии идеи или разработки, то выводится плановое значение; если процесс в эксплуатации, то фактическое. Например, значение `4,36` сообщает, что для выполнения процесса нужно 5 лицензий роботов, 4 из которых будут использоваться полностью, а одна частично.
* `Неавтоматизируемый человеческий труд` — количество человеческого труда, который не удалось автомтизировать. Показатель выводит усредненное значение за месяц, рассчитанное в течение всего срока эксплуатации.


</details>

Каждая карточка процесса имеет элементы управления в виде иконки **карандаша**:
* **Карандаш** в правом верхнем углу карточки показывает возможные действия с процессом. Так, пункт **Изменить** открывает страницу редактирования процесса — здесь вы можете изменить значения полей, добавить контакты и т.д. Действие **Удалить** безвозвратно удаляет карточку процесса. 
* **Карандаш** напротив статуса позволяет изменить текущее состояние процесса.

![](<../../idea-hub/resources/getting-started/process-management-icon.png>)

Если нажать на карточку процесса, то откроется дашборд с набором информационных панелей, где собрана полная информация о процессе. Как и в случае с дашбордом на главной странице, каждая панель гибко редактируется, а их набор можно сократить или расширить.

![](<../../idea-hub/resources/getting-started/process-detail.png>)

**Фильтрация процессов**

Помимо строки поиска, вы также можете использовать фильтры, которые отображаются в меню слева. Все выбранные фильтры будут суммироваться, в том числе и со значением, которое вы ввели в поле поиска.

![](<../../idea-hub/resources/getting-started/processes-filters.png>)



### Системы

На странице представлены все системы, которые участвуют в процессах роботизации. Системы могут быть созданы вручную или импортированы — для этого в файле `process.xlsx` предназначена колонка systems.

![](<../../idea-hub/resources/getting-started/systems.png>)

При нажатии на название системы в списке откроется страница процессов, на которой будут перечислены только те процессы, в которых задействована данная система.

Для добавления к процессу дополнительной системы, нажмите на кнопку редактирования процесса (*карандаш*) в правом верхнем углу карточки процесса и выберите **Изменить**. 
На открывшейся странице выберите вкладку **Основная информация >** поле **Системы**. По щелчку в данном поле откроется выпадающий список для выбора нужных систем. 

![](<../../idea-hub/resources/getting-started/add-system.png>)


### Очереди

Под очередями понимаются очереди обмена данными, информацию о которых Idea Hub получает при импорте данных из Оркестратора. В процессе импорта в Idea Hub переносятся все данные, относящиеся к данной очереди.

![](<../../idea-hub/resources/getting-started/queues.png>)

<details>

<summary>Описание таблицы очередей</summary>

* `Процесс` — процесс, с которым связана очередь.
* `Успех` — количество успешных выполнений элемента.
* `Исключения` — количество исключений, возникшие в ходе обработки элементов очереди. Под исключением понимается техническая нештатная ситуация, которая помешала работе робота.
* `Ошибка` — количество бизнес-ошибок.

</details>

Если нажать на название очереди, то откроется страница с детализированной информацией из Оркестратора. Цветная горизонтальная полоса внизу является графическим представлением статистики по данной очереди.

Внизу этой страницы также отобразится ежедневная статистика по обработке элементов очереди. Вы можете выбрать необходимый временной период и скачать эту статистику в файл `*.xls`.

![](<../../idea-hub/resources/getting-started/download-queue.png>)



### Лицензии

Страница содержит информацию по всем лицензиям, которые имеются у организации на текущий момент — как действующим, так и нет. Это могут быть лицензии для Оркестратора, роботов и Студии, сгруппированные по тенантам. Или лицензии из других внешних систем, которые администратор импортировал из соответствующего источника.

![](<../../idea-hub/resources/getting-started/licenses.png>)

В правой части таблицы визуализирован календарь, который подсветит красным, когда срок какой-либо лицензии будет близок к концу. Также эта информация будет присутствовать в виде текстового уведомления на главной странице. 

Вы можете фильтровать вывод лицензий по контурам и тенантам — соответствующие фильтры расположены в панели слева.

![](<../../idea-hub/resources/getting-started/licenses-filtres.png>)


### Роботы

Страница содержит информацию о роботах, данные о которых Idea Hub получает при импорте данных из Оркестратора. 

Вы можете фильтровать список роботов по их контурам и тенантам — соответствующие фильтры расположены в панели слева.

![](<../../idea-hub/resources/getting-started/robots.png>)

<details>

<summary>Описание таблицы роботов</summary>

* `Название` — имя робота в Оркестраторе.
* `Машина` — название физической или виртуальной машины, на которой развернут робот.

</details>

Если нажать на название робота или его машины, то откроется страница с подробными данными по выбранному объекту.

<details>

<summary>Данные робота</summary>

* `Ключ для расчёта` — параметр, который используется для доступа к свойствам сущности в формулах.
* `Идентификатор робота` — идентификатор робота в Оркестраторе.
* `Машина` — название физической или виртуальной машины, на которой развернут робот. Соответствует названию машины в Оркестраторе.
* `Тип лицензии` — тип лицензии, которую использует робот.
* `Папка` — название личной папки в Оркестраторе, к которой принадлежит робот.
* `Версия` — версия робота.
* `Архитектура` — разрядность робота.
* `Контур` — изолированные подсети внутри общей сети организации. Например, продуктивный контур может иметь название `Prod`, которое будет указанно в данном поле.
* `Тенант` — название тенанта, к которому принадлежит робот в Оркестраторе.

</details>

### Дашборды

Дашборд — это инструмент визуализации данных, который собирает и отображает ключевые показатели на одной странице. Он помогает быстро проанализировать информацию, используя графики, диаграммы и другие визуальные элементы. Например, дашборд может собирать и показывать экономические эффекты процессов, информацию по утилизации лицензий, статистику из Оркестратора, а также другие значимые показатели.

Пользователь без административных прав может только просматривать дашборды. Администратору доступна возможность создавать дашборды и выводить с их помощью любую необходимую информацию. 

**Как создать дашборд:**

1. Перейдите на страницу **Дашборды**.
2. Нажмите кнопку **Добавить дашборд** и укажите его название. Например, `Мой дашборд`.

   ![](<../../idea-hub/resources/getting-started/add-dashboard.png>)

3. Откроется пустая страница с возможностью добавлять информационные панели по кнопке `+` — нажмите ее.
4. Справа откроется меню для выбора информационных панелей, которые можно добавить в дашборд. Под каждой панелью дано краткое описание. Вы можете добавить столько панелей, сколько вам необходимо — для этого кликните на нужную панель.

   ![](<../../idea-hub/resources/getting-started/add-panel-dashboard.png>)

Каждая панель, которую вы добавили, гибко редактируется — можно изменить названия блоков (*контейнеров*) в панели, отображаемые индикаторы и порядок их вывода и т.д.

Например, вы можете создать дашборд со статистикой процесса по нескольким привязанным очередям. По умолчанию на странице **Процессы** выводится статистика только по первой привязанной очереди — если вам этого недостаточно, создайте собственный дашборд с формулой по всем очередям, задействованным в процессе.

![](<../../idea-hub/resources/getting-started/new-panel-in-dashboard.png>)




### Идеи

Страница содержит информацию об оформленных заявках (*идеях*) на роботизацию бизнес-процессов. 

Заявка создается по кнопке **Предложить идею** в нижней части функционального меню. 

![](<../../idea-hub/resources/getting-started/ideas.png>)


<details>

<summary>Описание таблицы с Идеями</summary>

* `Автор` — реальное имя пользователя Idea Hub, который предложил идею автоматизации бизнес-процесса. Если нажать на автора, то откроется карточка с детальной информацией о пользователе, например, его контактные данные.
* `Создано` — дата создания идеи.
* `Обновлено` — дата обновления идеи.
* `Название` — краткое название процесса, который пользователь указал при создании идеи. Редактируемое поле.
* `Процесс` — ссылка **Смотреть процесс** ведет на страницу процесса, связанного с заявкой на роботизацию. 
* `Заключение` — ссылка для создания заключения по оформленной заявке или, если заключение уже выдано, ссылка для просмотра деталей заключения.
* `Статус заключения` — текущий статус заключения по заявке. Возможные статусы: На рассмотрении; Одобрено; Отказано.

</details>
