# Обзор интерфейса

В статье представлена информация об основных функциональных элементах в веб-интерфейсе Idea Hub. Статья не ставит перед собой цель полностью описать все возможные настройки и функции каждого раздела.


## Меню

При входе в Idea Hub открывается главная страница. В зависимости от уровня доступа к данным, установленного для конкретного пользователя, внешний вид страницы может различаться.

С левой стороны экрана расположены панели меню:

![](<../../idea-hub/resources/getting-started/admin-menu.png>)

**Административное меню** (1) отображается только пользователю с ролью администратора. Оно предназначено для настройки продукта под нужды конкретной организации. Например, через него можно настроить конфигурацию, создать пользователей, назначить права доступа к данным, установить необходимые расширения. Поскольку это меню не всегда необходимо использовать, внизу находится кнопка, по которой его можно быстро свернуть и таким образом расширить рабочую область. 

**Пользовательское меню** (2) обеспечивает доступ пользователей к функциональным возможностям продукта. Более подробное описание каждого из пунктов меню приведено ниже.



### Главная

На главной странице по умолчанию отображается дашборд с информационными панелями, где собраны и отображены ключевые данные об автоматизации в организации. Они помогают оценить ситуацию в целом, без детализации по отдельным процессам или департаментам.

Кроме того, Idea Hub позволяет переопределить главную страницу — администратор может назначить любую страницу в качестве главной. Настройка устанавливается в административном меню: **Конфигурация > Система > Основные настройки сайта > поле Главная страница**. 

![](<../../idea-hub/resources/getting-started/main.png>)

Дашборд главной страницы отображает предзаданные панели от команды Primo RPA. Для каждой панели существуют инструменты для гибкой настройки — вы можете изменить метрики и формулы расчета, удалить панель, переместить вверх-вниз или добавить новую панель к уже существующим.

**Цифрами на рисунке отмечены:**

<details>

<summary>1 — Панель с обобщенными данными</summary>

* `Всего Ч/М` — какое количество человеко-месяцев было сокращено роботами за весь период работы роботов.
* `Фактический эффект` — экономический эффект в рублях, который организация получила в результате замены человеко-месяцев автоматизацией. 
* `Всего успешных операций` — суммарное количество операций, которые роботы выполнили за все время эксплуатации. Операция в данном случае — это 1 экземпляр процесса, который выполнил робот без ошибок. Данные по успешным операциям поступают из Оркестратора.

</details>

<details>

<summary>2 — Панель с экономическими показателями за год и общей информацией о процессах</summary>

**Первая карточка**:
* `Эффект в деньгах за текущий год` — фактическая экономия в рублях за текущий год, которую организация получила от сокращения человеко-месяцев. Рядом отображается дельта, которая показывает разницу относительно предыдущего года.
* `Эффект Ч/М за год` — фактическое количество человеко-месяцев за год, которое было сокращено при помощи автоматизации.
* `Утилизация лицензий за год` — отражает, насколько эффективно использовались лицензии роботов за год. Высокая утилизация может свидетельствовать о том, что лицензии используются эффективно, тогда как низкая может указывать на избыточные расходы. Данные по утилизации собираются из Оркестратора и выводятся в виде двух показателей:
   *  Коэффициент утилизации — количество дней в году, в течение которых использовались лицензии роботов. Например, у вас было 10 лицензий, длительность каждой из которых составляла 1 год. Значит 100% времени для 10 лицензий составляет 3650 дней. Если бы все лицензии использовались 24 часа в сутки, то данный показатель бы отразил значение `3650`.
   *  Процент — отношение фактического использования лицензий к общему количеству доступных лицензий. Например, если у вас есть 10 лицензий, и в течение года использовалось 8 из них, то утилизация лицензий составит 80%. 

**Вторая карточка**:
* `Всего процессов в эксплуатации` — сколько процессов на текущий момент находится в статусе эксплуатации. 
* `Всего автоматизируемых систем` — общее количество внешних систем, которые были задействованы в процессах. Например, это такие системы, как 1С, SAP и т.д.
* `Всего автоматизируемых отделов` — общее количество департаментов, в которых используются процессы в статусе эксплуатации. 

**Воронка автоматизации** визуально отражает, на каких стадиях находятся все имеющиеся процессы в организации:
* `Заявки` — количество процессов, которые находятся на стадии предложенных Идей.
* `В разработке` — количество процессов в разработке на текущий момент.
* `В проде` — количество процессов в эксплуатации на текущий момент.

</details>


<details>

<summary>3 — Панель с эффектами в разрезе временных периодов</summary>

* `Эффект от роботизации в квартал` — количество сокращенных трудозатрат за завершенный квартал. Ниже отображается сравнительный показатель по отношению к предыдыдущему кварталу.

</details>

</details>




### Департаменты

При нажатии на пункт меню **Департаменты** открывается страница с перечнем департаментов организации. По умолчанию отображаются только те департаменты, с которыми связаны какие-либо процессы. 

При включении параметра **Показать департаменты без процессов** также отобразятся департаменты, с которыми пока не связан ни один процесс.

![](<../../idea-hub/resources/departments.png>)

Для каждого департамента доступна детализированная информация о связанных процессах и рассчитанных экономических эффектах. Чтобы увидеть эту информацию, нажмите на название нужного департамента.

Например, на рисунке ниже отображены процессы для департамента **Безопасность и аудит**. Вы можете перейти в нужный процесс департамента прямо с этой страницы — просто выберите карточку процесса.

![](<../../idea-hub/resources/dep-inside.png>)

#### Добавление департаментов

Департаменты могут быть добавлены в Idea Hub как с помощью импорта файла `areas.xlsx`, так и вручную через административное меню или Active Directory.




### Процессы

На странице **Процессы** собрана информация обо всех роботизированных процессах в организации и рассчитанных для них показателях. 

![](<../../idea-hub/resources/getting-started/processes-page.png>)

**Цифрами на рисунке отмечены:**

<details>

<summary>1 — Кнопки, которые отвечают за представление процессов на странице </summary>

По умолчанию процессы отображаются в виде карточек, но вы можете переключиться на табличное представление. При выборе кнопки **В эксплуатации** отобразятся только процессы со статусом `Эксплуатация`.

</details>


<details>

<summary>2 — Строка для поиска процессов по текстовым полям</summary>

Для корректной работы поиска необходимо указывать слово полностью. 

</details>

<details>

<summary>3 — Карточки автоматизируемых процессов </summary>

Карточка процесса по умолчанию содержит следующие поля:
* `Департамент` — отдел, к которому относится процесс.
* `Системы` — внешние системы, которые задействованы в автоматизации процесса. Например, 1С, SAP и другие.
* `Статус` — текущий статус процесса. Статусы описывают жизненный цикл процесса внутри организации, начиная с этапа создания заявки на роботизацию (Идеация).
* `Оркестратор подключен` — параметр отображает, собирает ли процесс статистику из Оркестратора.

Под карточкой дополнительно отображаются показатели эффектов: 
* `Плановый FTE` — показывает сокращаемый FTE, которое рассчитал владелец процесса, когда подавал идею на роботизацию. 1 FTE — эквивалент полной занятности сотрудника в месяц или, другими словами, количество человеческого труда, которое сокращается за счет автоматизации.
* `Фактический FTE` — усредненный FTE в месяц за все время эксплуатации процесса. В скобках отображается дельта — насколько фактический FTE выше/ниже планового.
* `Утилизация лицензий` — плановый или фактический показатель утилизации лицензий для выполнения процесса. Значение зависит от статуса процесса: если процесс на стадии идеи или разработки, то выводится плановое значение; если процесс в эксплуатации, то фактическое. Например, значение `4,36` сообщает, что для выполнения процесса нужно 5 лицензий роботов, 4 из которых будут использоваться полностью, а одна частично. 
* `Неавтоматизируемый человеческий труд` — количество человеческого труда, который не удалось автомтизировать. (дописать)


</details>

Каждая карточка процесса имеет элемент управления. Нажмите на **карандаш** в правом верхнем углу карточки, чтобы отобразить возможные действия:

![](<../../idea-hub/resources/getting-started/edit-process-card.png>)

<details>

<summary>Доступные действия с процессом</summary>

Пункт **Изменить** открывает страницу редактирования полей процесса. Здесь вы можете изменить значения, переписать формулы для расчетов, добавить контакты и т.д.

Пункт **Удалить** позволяет удалить карточку процесса. Данное действие необратимо.

Также карточка процесса имеет редактируемое поле **Статус**. Если вы нажмете на **карандаш** возле статуса, то сможете изменить состояние процесса. //**Все статусы всегда только вручную меняются? Это основное место изменения статусов процесса?**

</details>

Чтобы просмотреть детализированную информацию о процессе, нажмите на его карточку. В результате откроется дашборд с набором информационных панелей — как и в случае с дашбордом на главной странице, каждая панель гибко редактируется, а их набор можно сократить или расширить.

![](<../../idea-hub/resources/getting-started/process-detail.png>)

**Фильтрация процессов**:

Помимо строки поиска, вы также можете использовать фильтры, которые отображаются в меню слева. Все выбранные фильтры будут суммироваться, в том числе и со значением, которое вы ввели в поле поиска.

### Системы

На странице представлены все системы, которые участвуют в процессах роботизации. 

![](<../../idea-hub/resources/getting-started/systems.png>)

Системы могут быть созданы вручную или импортированы — для этого в файле `process.xlsx` предназначена колонка systems.

При нажатии на название системы в списке откроется страница процессов, на которой будут перечислены только те процессы, в которых задействована данная система.

Для добавления к процессу дополнительной системы, нажмите на кнопку редактирования процесса (*карандаш*) в правом верхнем углу карточки процесса и выберите **Изменить**. 
На открывшейся странице выберите вкладку **Основная информация >** поле **Системы**. По щелчку в данном поле откроется выпадающий список для выбора нужных систем.

* `Процесс` — процесс, с которым связана очередь.

### Дашборды

Дашборд — это инструмент визуализации данных, который собирает и отображает ключевые показатели на одной странице. Он помогает быстро проанализировать информацию, используя графики, диаграммы и другие визуальные элементы. Например, дашборд может собирать и показывать экономические эффекты процессов, информацию по утилизации лицензий, статистику из Оркестратора, а также другие значимые показатели.

Idea Hub позволяет создавать любые дашборды и выводить с их помощью только нужную информацию. Для этого перейдите на страницу **Дашборды**, нажмите кнопку **Добавить дашборд** и укажите его название. Например, `Мой дашборд`.

![](<../../idea-hub/resources/getting-started/add-dashboard.png>)

Откроется пустая страница с возможностью добавлять информационные панели по кнопке `+`. При нажатии кнопки справа откроется меню для выбора информационных панелей, которые вы можете добавить в свой дашборд. Под каждой панелью дано краткое описание ее назначения. Вы можете добавлять столько панелей, сколько вам необходимо.

![](<../../idea-hub/resources/getting-started/add-panel-for-dashboard.png>)

Каждая панель, которую вы добавили, гибко редактируется — можно изменить названия блоков (*контейнеров*) в панели, отображаемые индикаторы и их порядок вывода, формулы расчетов индикаторов и другое.

![](<../../idea-hub/resources/getting-started/new-panel-in-dashboard.png>)

Например, вы можете создать дашборд со статистикой процесса по нескольким привязанным очередям. По умолчанию на странице **Процессы** выводится статистика только по первой привязанной очереди — если вам этого недостаточно, создайте собственный дашборд с формулой по всем очередям, задействованным в процессе.

### Идеи

Страница содержит информацию об оформленных заявках (*идеях*) на роботизацию бизнес-процессов.

![](<../../idea-hub/resources/getting-started/ideas.png>)


<details>

<summary>Колонки таблицы</summary>

* `Автор` — реальное имя пользователя Idea Hub, который предложил идею автоматизации бизнес-процесса. Если нажать на автора, то откроется карточка с детальной информацией о пользователе, например, его контактные данные.
* `Статус` — **текущий статус Идеи (??). Какие могут быть?** Отображается в виде иконки.
* `Создано` — дата создания идеи.
* `Обновлено` — дата обновления идеи **(тоже автором или любым пользователем?)**.
* `Название` — краткое название процесса, который пользователь указал при создании идеи. Редактируемое поле.
* `Процесс` — ссылка **Смотреть процесс** ведет на страницу процесса, связанного с заявкой на роботизацию. 
* `Заключение` — ссылка для создания заключения по оформленной заявке или, если заключение уже выдано, ссылка для просмотра деталей заключения.
* `Статус заключения` — текущий статус заключения по заявке. 

</details>

Заявка создается по кнопке **Предложить идею** в нижней части функционального меню. 


### Импорт данных

Страница содержит возможности для загрузки, импорта и экспорта файлов с данными о процессах, управлениях и департаментах, пользователях, которые необходимы для работы в Idea Hub. 

Система получает информацию из баз данных Оркестратора с помощью SQL-запросов, которые запускаются скриптом `orc-data-fetch` и формируются в файлы формата CSV. Сформированные CSV-файлы сохраняются в специальным образом организованной структуре каталогов, откуда автоматически импортируются собственными средствами Idea Hub.
