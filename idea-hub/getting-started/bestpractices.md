# Типы процессов

Данный раздел поможет написать RPA-роботов, с которыми может наиболее эффективно взаимодействать Idea Hub.

Существуют следующие типы процессов:
1. Транзакционный процесс (Enterprise license).
1. Линейный процесс (Enterprise license).


## Транзакционный процесс
Транзакционный процесс — это процесс, в ходе которого поступающая информация разделяется на одинаковые запросы. Для каждого запроса выполняется своя логика и обработка.

### Транзакция — это элемент очереди

Робот собирает информацию из информационной системы и записывает каждую транзакцию в очередь. В итоге можно получить необходимую для расчетов в Idea Hub информацию, сколько раз процесс выполнялся (или сколько было попыток его выполнить) и сколько раз он выполнялся успешно.

Другим важным параметром, также используемым в расчетах, является время обработки элемента очереди.

Таким образом можно посчитать:  
*	Среднее время работы каждой транзакции.  
*	Количество транзакций за определенный период.  
*	Утилизацию лицензий для конкретного процесса.  
*	Окупаемость данного процесса. – **!!!спросить у К, как считается**  
*	Статистика по успешным / ошибочным транзакциям.  
*	Суммарный эффект по процессу в человеко-месяцах – ск труда робот сэкономил, в чел-месяцах – **!!!подробности узнать у К**  
*	Суммарный эффект по процессу в рублях.

### Транзакция — это строка в Excel-файле (офисные файлы) или SQL в базе данных

Робот собирает информацию из источника данных и обрабатывает построчно по условиям.

Тем самым можно посчитать:  
*	Среднее время работы всего робота.  
*	Утилизацию лицензий для конкретного процесса.  
*	Окупаемость данного процесса.  
*	Статистика по успешным / ошибочным запускам.  
*	Может что-то еще?

## Линейный процесс

Линейный процесс — это процесс, который представляет собой последовательный алгоритм однократной обработки данных. 

Тем самым мы можем посчитать:  
*	Среднее время работы всего робота.  
*	Утилизацию лицензий для конкретного процесса.  
*	Окупаемость лицензии данного процесса.  
*	Статистика по успешным / ошибочным запускам.  
*	Статистика по успешным / ошибочным запускам.  
*	Может что-то еще?
