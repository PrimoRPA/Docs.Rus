# Главная страница

При входе на сайт Idea Hub открывается главная страница. В зависимости от уровня доступа к данным, установленного для конкретного пользователя, внешний вид страниц может различаться.


## Меню

С левой стороны расположены административное меню и функциональное меню:

1 — **Административное меню** предназначено для настройки продукта под нужды конкретной организации. Например, в нем можно настроить конфигурацию, создать пользователей, назначить права доступа к данным.

2 — **Функциональное меню** обеспечивает доступ к функциональным возможностям продукта. Более подробное описание каждого из пунктов приведено ниже.

![](<../../idea-hub/resources/main-admin-menu.png>)

### Департаменты

При нажатии на пункт меню **Департаменты** открывается страница с перечнем департаментов организации. По умолчанию отображаются только те департаменты, с которыми связаны какие-либо процессы. 

При включении параметра **Показать департаменты без процессов** также отобразятся департаменты, с которыми пока не связан ни один процесс.

![](<../../idea-hub/resources/departments.png>)

Департаменты могут быть добавлены в Idea Hub как с помощью импорта файла `areas.xlsx`, так и вручную через Административное меню или Active Directory.

Чтобы добавить департамент через Административное меню, перейдите в раздел **Структура > Таксономия > Департаменты/Управления > Список терминов**:

![](<../../idea-hub/resources/add-departments-by-menu.png>)

И выберите **Добавить термин**:

![](<../../idea-hub/resources/add-departments-by-menu-button.png>)

Форма создания департамента содержит следующие элементы:
* Имя\* — название департамента.
* Описание — произвольный текст, описание департамента.
* Код — данное поле используется для связи с процессами во время импорта. Именно указанный в данном поле код должен быть использован в файле `process.xlsx` для корректного связывания департамента с относящимися к нему процессами.
* Иконка — здесь можно загрузить изображение, которое будет использоваться для данного департамента в списке на странице **Департаменты**.
* Отношения — если добавляемый департамент находится в подчинении у другого, выберите из данного списка вышестоящий (*родительский*) департамент. Иерархия будет отражена в списке на странице **Департаменты**.
* Вес\* — департаменты упорядочиваются сначала по весу, а затем по названию. По умолчанию задан вес `0`.

Добавление департаментов через Active Directory требует предварительного подключения LDAP и настройки в соответствии со спецификой конкретной организации.

Для каждого из департаментов в списке доступна страница с индикаторами, отображающими рассчитанные параметры и процессы, существующие для данного департамента. 

При нажатии на названии департамента откроется его страница, содержащая статистику, а также карточки относящихся к данному департаменту процессов.

![](<../../idea-hub/resources/dep-inside.png>)


### Процессы

На странице Процессы отображается информация о роботизированных процессах организации и рассчитанных для них показателях.

![](<../../.gitbook/assets1/idea-hub/>)

В верхней части страницы процесса расположено поле **Поиск**:

![](<../../.gitbook/assets1/idea-hub/>)

Для поиска по текстовым полям введите в это поле слово для поиска.

:large_orange_diamond:***Внимание.** для корректной работы поиска необходимо указывать слово полностью.*

Для результатов поиска может быть настроена приоритетность. При этом в выдаче будет учитываться, в каких именно полях найдено искомое слово: в названии процесса, описании, списке систем и т.д.

В нижней левой части экрана расположены фильтры:

![](<../../.gitbook/assets1/idea-hub/>)

С их помощью можно проводить дальнейшую гибкую фильтрацию результатов поиска.

Внешний вид карточек процессов можно настроить, как и содержащуюся в них информацию (например, можно изменить как набор отображаемых полей, так и формулы расчета и т.п.). 

Для редактирования процесса нажмите на изображении ручки в правом верхнем углу карточки, затем выберите **Изменить** — откроется страница редактирования процесса:

![](<../../.gitbook/assets1/idea-hub/>)

Также каждая карточка процесса имеет редактируемое поле Статус. При нажатии на значок **ручка** в этом поле будет открыта страница редактирования статуса. 

![](<../../.gitbook/assets1/idea-hub/>)

Статусы процесса описывают его жизненный цикл внутри организации начиная с этапа создания заявки на роботизацию (этап Идеация). 

!! При указании для процесса статуса “Остановлен” он будет продолжать учитываться в расчетах показателей эффективности за предыдущий период.

### Системы

Здесь перечислены системы, задействованные в организации, которые участвуют в процессах роботизации. 

![](<../../.gitbook/assets1/idea-hub/>)

Системы могут быть созданы вручную или импортированы — для этого в файле `process.xlsx` предназначена колонка systems.

При нажатии на названии системы в списке откроется страница процессов, на которой будут перечислены только те процессы, в которых задействована данная система.

Для добавления к процессу дополнительной системы, нажмите на кнопку редактирования процесса (“ручка”) в правом верхнем углу карточки процесса и выберите **Изменить**. 
На открывшейся странице выберите вкладку **Основная информация >** поле **Системы**. По щелчку в данном поле откроется выпадающий список для выбора нужных систем. 

![](<../../.gitbook/assets1/idea-hub/>)


### Очереди

В данном контексте под очередями понимаются очереди обмена данными, информацию о которых Idea Hub получает при импорте данных из Оркестратора. 

![](<../../.gitbook/assets1/idea-hub/>)

В процессе импорта в Idea Hub переносятся все данные, относящиеся к данной очереди.

Колонки таблицы:
* Успех — количество успешных выполнений элемента.
* Исключения — количество исключений (технических нештатных ситуаций, помешавших работе робота), возникшие в ходе обработки элементов очереди.
* Ошибки — количество бизнес-ошибок.

При нажатии на названии очереди откроется страница с информацией о ней, содержащая данные, полученные из Оркестратора, а также круговую диаграмму со статистикой обработки очереди. Цветная горизонтальная полоса внизу также является графическим представлением статистики по данной очереди.

![](<../../.gitbook/assets1/idea-hub/>)


### Роботы

Страница содержит информацию о роботах, данные о которых Idea Hub получает при импорте данных из Оркестратора, включая статистику по запускам в целом и успешным запускам.

![](<../../.gitbook/assets1/idea-hub/>)

В левой верхней части страницы расположено поле “Тенант” с выпадающим списком, с помощью которого можно отфильтровать роботов по их тенантам.

При нажатии на названии робота откроется страница с данными конкретного робота, такими как машина робота, тип используемой лицензии, версия и т.д.:

![](<../../.gitbook/assets1/idea-hub/>)


### Лицензии

При выборе данного пункта меню откроется страница “Лицензии”, содержащая информацию по имеющимся на текущий момент лицензиям (как действующим, так и недействующим) для Оркестратора, Роботов и Студии, сгруппированную по тенантам.

![](<../../.gitbook/assets1/idea-hub/>)

Используя фильтры в верхней части страницы и флажок “Только активные” информацию можно отфильтровать по типам лицензий, тенантам и по активности лицензий.

Если срок действия какой-либо лицензии подходит к концу, это будет визуально отображаться в календаре справа в виде полосы красного цвет напротив соответствующего типа лицензий. Также эта информация будет присутствовать в виде текстового уведомления на странице. Дополнительно можно настроить отправку данного уведомления по электронной почте.


### Машины

![](<../../.gitbook/assets1/idea-hub/>)

При нажатии на названии машины робота в списке будет открыта страница данной машины.

![](<../../.gitbook/assets1/idea-hub/>)

На этой странице также содержится статистика по роботам, выполняемым на данной машине.

![](<../../.gitbook/assets1/idea-hub/>)

### Идеи

Страница “Идеи” содержит информацию об оформленных заявках на роботизацию.

![](<../../.gitbook/assets1/idea-hub/>)

При нажатии на ссылку **Смотреть процесс** в колонке “Процесс” будет открыта страница процесса, связанного с данной заявкой на роботизацию, где отражена основная информация о нем.

Нажав на надпись в колонке “Заключение” можно посмотреть имеющееся заключение по конкретной заявке (если заключение было создано, ссылка будет иметь вид “Смотреть заключение”), либо создать заключение, если оно отсутствует (ссылка **Создать заключение**).

Ссылки в колонке “Автор” ведут на страницы с информацией о создателях заявок на роботизацию, включая их контактные данные.

Более подробную информацию об оформлении заявок на роботизацию можно найти на странице “Создание идеи” [link].
