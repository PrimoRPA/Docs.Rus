# Файл process.xlsx

Файл `process.xlsx` позволяет массово добавить в Idea Hub информацию об автоматизированных бизнес-процессах. Данный файл заполняется вручную и затем загружается на странице **Импорт данных**. Заполненный файл должен называться `process.xlsx` — не меняйте его название и расширение.

Скачать шаблон файла можно по [ссылке](https://github.com/PrimoRPA/Docs.Rus/tree/main/file-for-download) либо в веб-интерфейсе Idea Hub на странице **Импорт данных** по кнопке **Получить образец файла**. Данная страница доступна только пользователю с административными правами.

![](<../../../../idea-hub/resources/installation/import-get-sample-file.png>)


## Описание полей файла process.xlsx

Поля, отмеченные “звездочкой” (*), являются обязательными для заполнения. Данные вносятся в файл в соответствии с правилами и форматом, указанными в таблице ниже.

**Минимальный набор полей, которые должны быть в наличии для создания полноценной карты процесса — id, title, dep, systems, auto_prc, transactions, hum_duration, hum_cost, launched, workflow.**


{% hint style="info" %}
**Примечание**. Возможность Idea Hub рассчитывать показатели зависит от наличия информация в полях данного файла. Хотя большинство полей не являются обязательными, если они не будут заполнены, многие показатели рассчитать будет невозможно. Чем меньше полей заполнено, тем меньше показателей Idea Hub может рассчитать. 
{% endhint %}

{% hint style="warning" %}
**Важно.** Для правильной обработки данных, представляющих собой календарные даты, ячейки таблицы, в которых сохраняются подобные данные, должны иметь формат "Дата".
{% endhint %}

{% hint style="warning" %}
**Важно.** Для корректной работы с файлом, не рекомендуется использовать в ячейках файла какие-либо формулы (за исключением полей effect_actual и effect_planned, в которых могут быть использованы формулы Idea Hub). Информацию по синтаксису формул в Idea Hub можно найти [здесь](https://docs.primo-rpa.ru/primo-rpa/primo-rpa-idea-hub/about/formulas).
{% endhint %}


| **Название поля** | **Описание** | **Формат** | 
| --- | --- | --- |
| id* | Обязательное поле.  Уникальный идентификатор процесса, любая строка. Рекомендуемая длина - 10 символов. | Текст |
| title* | Обязательное поле.  Короткое описание процесса. | Текст |
| desc | Подробное описание. | Текст |
| dep* | Код управления/департамента.  Код должен соответствовать одному из значений поля «code» в файле areas.xlsx. | Текст |
| systems* | Список систем и программ, с которыми работает проект; можно указать несколько значений, разделенных символом \| (вертикальная черта). Системы будут создаваться автоматически при загрузке. | Текст |
| auto\_prc* | % транзакций процесса, поддающихся автоматизации. Данный показатель - это отношение общего количества успешных транзакций к общему количеству транзакций выраженное в процентах. | Число |
| fte | Количество оптимизируемых процессом FTE. Используется, только если невозможно рассчитать по другим данным. Если поле не заполнено, то его значение будет автоматически рассчитано, исходя из данных, предоставляемых в выгрузке process.xlsx.| Число |
| hum\_duration* | Скорость выполнения одной транзакции процесса ЧЕЛОВЕКОМ (в минутах). | Число |
| duration | Скорость выполнения одной транзакции процесса РОБОТОМ (в минутах). | Число |
| transactions* | Значение, указывающее на то, сколько раз выполняется бизнес операция за указанный в поле "transactions_period" период. | Число |
| transactions_period | Это поле связано с полем "transactions" и указывает, за какой период берутся значения для поля "transactions". | Возможные варианты значения этого поля: hour, day, week, month, quarter, year. |
| hum\_cost* | Стоимость 1 FTE на процессе. Указывается месячная заработная плата "gross" одного сотрудника. Также можно указать месячную заработную плату "net" одного сотрудника, а в расчётах для приведения размера зарплаты до уровня "gross" использовать произвольный коэффициент, рассчитанный для вашей компании. Как завести коэффициент и использовать его в расчётах можно прочитать в описании [параметров расчетов](https://docs.primo-rpa.ru/primo-rpa/primo-rpa-idea-hub/about)  | Число |
| owner | Email пользователя IH (владельца процесса). Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Email |
| contacts | Email пользователей в IH. Здесь могут быть указаны контакты пользователей, которые имеют отношение к процессу, но не являются его разработчиками или владельцами. Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Email |
| launched* | Дата старта эксплуатации процесса, в формате dd.mm.yyyy | Дата |
| finished | Дата вывода процесса из эксплуатации. | Дата |
| dev\_started | Дата начала разработки. | Дата |
| schedule | Расписание, в какие часы робот может запускать этот процесс. Часы дня от 0 до 23, разделенных символом \| (вертикальная черта), минуты всегда в формате 00, т.е. например 00:00\|02:00\|04:00 | Текст в формате «00:15\|00:30\|19:40» |
| complexity | Не используется | |
| type | Произвольное название типа автоматизации, например “Оптимизация бизнес-процесса” или “Сокращение FTE”. Это поле носит информационный характер и позволяет использовать значения, указанные в нем, для фильтрации процессов. Кроме того, потенциально это поле может быть использовано в формулах, для изменения логики расчёта в зависимости от указанного значения. | Текст |
| workflow* | Очень важная колонка, список возможных значений: process\_1\_ideation; process\_2\_design; process\_3\_development; process\_4\_testing; process\_5\_deployment; process\_6\_production; process\_7\_stopped; process\_8\_rejected. | Текст |
| queue\_id | GUID из Оркестратора или номер очереди (любое уникальное значение, связанное с очередью в Оркестраторе). Можно указать несколько значений, разделённых символом \| (вертикальная черта). | Guid |
| effect\_planned | Плановый эффект, в млн. рублей. Может быть любым значением или формулой Idea Hub. Данный показатель используется в расчетах. | robot\_cost | Стоимость разработки робота, в млн. рублей. | Число |
| robot\_type | Тип лицензии (возможные значения: Desktop, Enterprise, Miner, Orchestrator, Robot Standard, Studio, Testing) | Текст |
| robot\_user | Email пользователей (владельцев процесса) в IH. Можно указать несколько значений, разделенных символом \| (вертикальная черта).  | Email |
| machine\_name | Имя машины, как оно выгружается в файле выгрузки machines.csv в поле «name». Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Текст |
| responsible\_analyst | Email пользователей (аналитиков) в IH. Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Email |

| other | Ссылка на файл, который нужно загрузить и прикрепить к документу. В данном поле можно указать ссылки на те документы, которые должны сопровождать  процесс (например, протокол о вводе в ПЭ). Формат файла: «Название файла,URL файла,\|Название файла 2,URL файла 2». | Ссылка на файл |

Также поддерживается использование символа-маски \* (звездочка), заменяющего собой произвольное количество символов в начале или в конце имени (в качестве префикса либо суффикса), для одномоментного указания проектов с похожими или шаблонными значениями имен. ## Импорт файла 

Заполненный файл `process.xlsx` следует загрузить в веб-интерфейсе Idea Hub на странице **Импорт данных**. Пользователю потребуется права администратора.

Для импорта данных:
* В строке **Процессы** нажмите на стрелочку рядом с действием **Импортировать**.
* Выберите пункт **Загрузить файл**.
* Выберите файл для загрузки и нажмите **Загрузить файл**.
* Затем нажмите кнопку **Импортировать**.
