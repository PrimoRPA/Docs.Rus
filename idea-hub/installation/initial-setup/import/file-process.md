# Файл process.xlsx

С помощью файла `process.xlsx` вы можете массово импортировать в Idea Hub информацию об автоматизированных бизнес-процессах. Это позволит не создавать каждый процесс вручную и таким образом, сократить время для добавления данных в Idea Hub. 

Скачать шаблон файла можно в веб-интерфейсе Idea Hub на странице **Импорт данных**. Данная страница доступна только пользователю с административными правами. Чтобы скачать шаблон, нажмите кнопку **Получить образец файла**.

![](<../../../../idea-hub/resources/installation/import-get-sample-file.png>)

Шаблон содержит предустановленный набор полей процесса, который можно расширить в зависимости от потребностей организации. Добавление новых полей процесса выполняется только через веб-интерфейс Idea Hub — если вы добавите новые поля в сам шаблон, то изменения не применятся. 

Файл шаблона, доступный по кнопке **Получить образец файла**, автоматически обновляется. Это значит, что если вы добавите в тип материала **Процесс** новые поля для импорта, то шаблон файла обновится и будет содержать эти поля.

## Перед началом импорта

1. Изучите шаблон `process.xlsx`. Если полей процесса недостаточно, добавьте их через веб-интерфейс согласно этой [инструкции](https://docs.primo-rpa.ru/primo-rpa/primo-rpa-idea-hub/processes/add-fields).
1. Для поля, который вы хотите добавить в шаблон, обязательно должна быть установлена настройка **Включить импорт/экспорт поля**.
1. После добавления необходимых вам полей скачайте обновленный шаблон по кнопке **Получить образец файла** и заполните его в соответствии с правилами и форматом, указанными в таблице ниже.
1. Заполненный файл должен называться `process.xlsx` — не допускается менять его название и расширение.




## Описание полей файла process.xlsx

Информация по заполнению полей шаблона приведена в таблице ниже, а также содержится в самом шаблоне в виде подсказок. Чтобы их увидеть, наведите курсор на название нужного поля.

**Обязательные поля для заполнения — id, title, area, environment, workflow, tenant. Однако для создания полноценной карты процесса рекомендуем также заполнить поля — systems, automation_prc, transactions, human_duration, fte_human_fte_cost, production_since.**


{% hint style="info" %}
**Примечание**. Возможность Idea Hub рассчитывать показатели зависит от наличия информация в полях данного файла. Хотя большинство полей не являются обязательными, если они не будут заполнены, многие показатели рассчитать будет невозможно. Чем меньше полей заполнено, тем меньше показателей Idea Hub может рассчитать. 
{% endhint %}


{% hint style="warning" %}
**Важно.** Для корректной работы с файлом, не рекомендуется использовать в ячейках файла какие-либо формулы (за исключением полей effect_actual и effect_planned, в которых могут быть использованы формулы Idea Hub). Информацию по синтаксису формул в Idea Hub можно найти [здесь](https://docs.primo-rpa.ru/primo-rpa/primo-rpa-idea-hub/about/formulas).
{% endhint %}

| id                | Да                 | Уникальный идентификатор процесса, любая строка. Рекомендуемая длина — 10 символов | Текст |
| title             | Да                 | Название процесса           | Текст |
| body              | Нет                | Краткое описание процесса   | Текст |
| tenant            | Да                 | Тенант. Название тенанта, к которому относится процесс в Оркестраторе. Тенант — логический объект Оркестратора, который соответствует филиалу организации и включает совокупность пользователей и проектов, хранящих наборы данных | Текст |
| area              | Да                 | Департамент. В значении следует указать код департамента организации, для которого разрабатывался процесс. Код должен соответствовать одному из значений поля code в файле `areas.xlsx` | Текст |
| automation_prc    | Нет                | Автоматизация. Потенциальный процент автоматизации бизнес-процесса, который рассчитал владелец процесса при его создании | Число |

Если статистики по процессу нет, то в этом поле можно указать число. <p>Если поле не заполнено, то его значение будет автоматически рассчитано по формуле по умолчанию, исходя из данных, предоставляемых в выгрузке `process.xlsx` </p> | Формула |

Если значение в поле указано в виде числа, то оно будет рассматриваться, как эффект от данного процесса, выраженный в рублях | Формула |

| ops_period        | Нет                | Период операций. Это поле указывает, за какой период берется значение для поля `transactions` | Текст. Возможные значения: hour, day, week, month, quarter, year |
| fte_human_fte_cost | Да                | Стоимость FTE в месяц. Стоимость рабочего месяца сотрудника (gross), который выполняет данный процесс. Также можно указать месячную заработную плату "net" одного сотрудника, а в расчетах для приведения размера зарплаты до уровня "gross" использовать произвольный коэффициент, рассчитанный для вашей компании. Как завести коэффициент и использовать его в расчетах можно прочитать в описании [параметров расчетов](https://docs.primo-rpa.ru/primo-rpa/primo-rpa-idea-hub/about) | Число |
| dev_cost          | Нет                | Стоимость разработки. Стоимость разработки автоматизированного процесса | Число |
| dev_started       | Нет                | Дата начала разработки | ДД.ММ.ГОД. Пример: `16.05.2024` |
| production_since  | Нет                | Дата начала. Дата ввода процесса в эксплуатацию, которая используется для расчета эффектов | ДД.ММ.ГОД. Пример: `16.05.2024` |
| production_until  | Нет                | Вывод процесса из эксплуатации. Дата планового вывода процесса из эксплуатации | ДД.ММ.ГОД. Пример: `16.05.2024` |
| workflow          | Да                 | Статус жизненного цикла процесса. Список возможных значений по умолчанию: <p> process\_1\_ideation; (Идеация) <br> process\_2\_design; (Проектирование) <br> process\_3\_development; (Разработка) <br> process\_4\_testing; (Тестирование) <br> process\_5\_deployment; (Развертывание) <br> process\_6\_production; (Экплуатация) <br> process\_7\_stopped; (Остановлен) <br> process\_8\_rejected (Отклонён) </p> <p> Импортер пропустит строку с процессом, если значение Workflow не совпадает ни с одним из допустимых значений</p>| Текст |
| schedule          | Нет                | Расписание. Интервалы времени, в которые робот может запускать процесс. Указываются часы дня от 0 до 23, разделенных символом \| (вертикальная черта), минуты всегда в формате 00, т.е. например 00:00\|02:00\|04:00 | Текст в формате «00:15\|00:30\|19:40» |
| queue_ui          | Нет                | Имена очередей. Название очереди обмена данными (или часть названия) из Оркестратора. Значение регистрозависимое. Допускается несколько значений, разделённых символом \| (вертикальная черта) | Текст |

Поле тенант добавить в файл - оно обязательное. Обязательные - только id, title, area, environment, workflow, tenant. Все остальные необязательные.

В глоссарий:  * транзакция - в контексте ИХ это набор операций, выполненных системой в рамках одной итерации бизнес-процесса. Совокупность нескольких операций, связанных с одной итерацией бизнес-процесса - например, обработка одного элемента очереди.



## Импорт файла process.xlsx

Для импорта требуются права администратора. Чтобы импортировать данные о процессах:
1. Перейдите в веб-интерфейс Idea Hub на страницу **Импорт данных**
1. В строке **Процессы** нажмите на стрелочку рядом с действием **Импортировать**.
1. Выберите пункт **Загрузить файл**.
1. Выберите заполненный файл `process.xlsx` и нажмите **Загрузить файл**.
1. Затем в строке **Процессы** нажмите кнопку **Импортировать**. Информация из ранее загруженных файлов `users.xlsx` и `areas.xlsx` добавится автоматически при импорте процессов.
