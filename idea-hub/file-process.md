# Файл process.csv

Файл, который будет отправлен для импорта, должен быть сохранён с именем **process** и в формате **.xlsx**.  
Руководство по сохранению файла в вашем редакторе в нужном формате можно найти либо в справочной системе редактора, либо в Интернете.  

Данные вносятся в файл в соответствии с правилами и форматом, указанными в таблице ниже.

**‼️ВАЖНО‼️ Для правильной обработки данных, представляющих собой календарные даты, ячейки таблицы, в которых сохраняются подобные данные, должны иметь формат "Дата".**

**‼️ВАЖНО‼️ Для дальнейшей корректной работы с файлом, не рекомендуется использовать в ячейках файла какие-либо формулы.**

В комплекте документации поставляется шаблон файла process.xlsx - в него могут быть внесены данные, а затем файл необходимо сохранить с тем же именем (process.xlsx), не меняя имя и расширение файла.

Скачать шаблон файла также можно по [ссылке.](https://github.com/PrimoRPA/Docs.Rus/tree/main/file-for-download)


**Формат файла process.xlsx:**

| **Название поля** | **Описание** | **Формат** | 
| --- | --- | --- |
| id | Обязательное поле.  Уникальный идентификатор процесса, любая строка. Рекомендуемая длина - 10 символов. | Текст |
| title | Обязательное поле.  Короткое описание процесса. | Текст |
| desc | Подробное описание. | Текст |
| dep | Код управления/департамента.  Код должен соответствовать одному из значений поля «code» в файле areas.csv. | Текст |
| systems | Список систем и программ, с которыми работает проект; можно указать несколько значений, разделенных символом \| (вертикальная черта). Системы будут создаваться автоматически при загрузке. | Текст |
| auto\_prc | % транзакций процесса, поддающихся автоматизации. | Число |
| fte | Количество оптимизируемых процессом FTE. Используется, только если невозможно рассчитать по другим данным. Если поле не заполнено, то его значение будет автоматически рассчитано, исходя из данных, предоставляемых в выгрузке process.csv.| Число |
| hum\_duration | Скорость выполнения одной транзакции процесса ЧЕЛОВЕКОМ (в минутах). | Число |
| duration | Скорость выполнения одной транзакции процесса РОБОТОМ (в минутах). | Число |
| transactions | Количество транзакций процесса (сколько раз в день его надо выполнять). | Число |
| hum\_cost | Стоимость 1 FTE на процессе (стоимость рабочего дня сотрудника \* 1.8). 1,8 - коэффициент, который позволяет пересчитать зарплату сотрудника в стоимость для компании. В ваших расчётах этот коэффициент может быть другим.  | Число |
| owner | Email пользователя IH, владельца процесса. Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Email |
| contacts | Email пользователей-разработчиков в IH. Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Email |
| launched | Дата старта эксплуатации процесса, в формате dd.mm.yyyy | Дата |
| finished | Дата вывода процесса из эксплуатации. | Дата |
| dev\_started | Дата начала разработки. | Дата |
| schedule | Расписание, в какие часы робот может запускать этот процесс. Часы дня от 0 до 23, разделенных символом \| (вертикальная черта), минуты всегда в формате 00, т.е. например 00:00\|02:00\|04:00 | Текст в формате «00:15\|00:30\|19:40» |
| complexity | Не используется | |
| type | Тип автоматизации, произвольный, используется только для аналитики. Новые типы создаются автоматически при импорте. | Текст |
| workflow | Очень важная колонка, список возможных значений: process\_1\_ideation; process\_2\_design; process\_3\_development; process\_4\_testing; process\_5\_deployment; process\_6\_production; process\_7\_stopped; process\_8\_rejected. | Текст |
| queue\_id | GUID из Оркестратора или номер очереди (любое уникальное значение, связанное с очередью в Оркестраторе). Можно указать несколько значений, разделённых символом \| (вертикальная черта). | Guid |
| effect\_planned | Планируемый эффект, в млн. рублей. | Число |
| effect\_actual | Действительный эффект, в млн. рублей. | Число |
| robot\_cost | Стоимость разработки робота, в млн. рублей. | Число |
| project | Название проекта. | Текст |
| robot\_type | Тип лицензии. | Текст |
| robot\_user | Email пользователей разработчиков в IH. Можно указать несколько значений, разделенных символом \| (вертикальная черта).  | Email |
| machine\_name | Имя машины, как оно выгружается в файле выгрузки machines.csv в поле «name». Можно указать несколько значений, разделенных символом \| (вертикальная черта). | Текст |
| responsible\_analyst | Email пользователей-аналитиков в IH. Можно указать несколько значений,разделенных символом \| (вертикальная черта). | Email |
| responsible\_developer | Email пользователей разработчиков в IH. Можно указать несколько значений, разделенных символом \| (вертикальная черта).  | Email |
| other | Ссылка на файл, который нужно загрузить и прикрепить к документу. Формат файла: «Название файла,URL файла,\|Название файла 2,URL файла 2». | Ссылка на файл |
| project\_orch | Идентификаторы проектов в Оркестраторе, связанных с данным процессом - с их помощью осуществляется связь с данными из Оркестратора. Можно указать несколько идентификаторов, разделенных символом \| (вертикальная черта). Например, project1\|project2. Также поддерживается использование символа-маски \* (звездочка), заменяющего собой произвольное количество символов в начале или в конце идентификатора (в качестве префикса либо суффикса), для одномоментного указания проектов с похожими или шаблонными значениями идентификаторов. Например, для привязки к процессу проектов с идентификаторами project1, project2,..., project99, в поле таблицы можно указать project\*. Поле может одновременно содержать и символ \|\, и \*, например: project1\|workitem*.| Текст |
| contour | Контур, к которому относится процесс. Контур - это изолированная подсеть внутри сети организации. Если в этом поле ничего не указано, то по умолчанию будет использовано значение Prom. | Текст |
