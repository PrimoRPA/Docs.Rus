# Файл process.csv

Файл, который будет отправлен для импорта, должен быть сохранён обязательно с именем **process** и в формате **.csv**.  
Руководство по сохранению файла в вашем редакторе в нужном формате можно найти либо в справочной системе редактора, либо в Интернете.  
В качестве разделителя в файле должна использоваться табуляция. Значения ячеек обёрнуты в кавычки "строка".  

Обратите внимание, что при сохранении в формате **.csv** формулы не сохраняются. Вместо этого в ячейках сохраняются рассчитанные значения.

В комплекте документации поставляется шаблон файла process.csv - в него могут быть внесены данные, а затем файл необходимо сохранить с тем же именем (process.csv), не меняя имя и расширение файла и не добавляя в них никаких символов.

Скачать шаблон файла также можно по [ссылке.](https://github.com/PrimoRPA/Docs.Rus/tree/main/file-for-download)

**Формат файла process.csv:**

| **Название поля** | **Описание** | **Формат** |
| --- | --- | --- |
| id | Обязательное поле. Уникальный идентификатор процесса, любая строка. Рекомендуемая длина - 10 символов. | Текст |
 title | Длинное описание процесса | Текст |
| desc | Подробное описание | Текст |
| dep | Код управления/департамента. Код должен соответствовать одному из значений поля «code» в файле areas.csv. | Текст |
| systems | Список систем и программ с которыми работает проект, через символ вертикальной черты. Системы будут создаваться автоматически при загрузке | Текст |
| auto\_prc | % транзакций процесса, поддающихся автоматизации; | Число |
| fte | Кол-во оптимизируемых процессом FTE. Используется только если невозможно рассчитать другим данным. | Число |
| hum\_duration | Скорость выполнения одной транзакции процесса ЧЕЛОВЕКОМ (в часах в месяц) | Число |
| duration | Скорость выполнения одной транзакции процесса РОБОТОМ (в часах) | Число |
| transactions | Количество транзакций процесса (сколько раз в день его надо выполнять) | Число |
| hum\_cost | Стоимость 1 FTE на процессе (стоимость рабочего дня сотрудника \* 1.8) | Число |
| team | Название команды | Текст |
| owner | Email пользователя IH владельца процесса | Email |
| contacts | Email пользователей разработчиков в IH, через запятую «,» | Email |
| launched | Дата старта эксплуатации процесса, в формате dd.mm.yyyy | Дата |
| finished | Дата вывода процесса из эксплуатации | Дата |
| dev\_started | Дата начала разработки | Дата |
| schedule | Расписание, в какие часы робот может запускать этот процесс. Часы дня от 0 до 23, через запятую, минуты всегда в формате 00, т.е. например 00:00,02:00,04:00 | Текст в формате «00:15,00:30,19:40» |
| complexity | Не используется | |
| type | Тип автоматизации, произвольный, используется только для аналитики | Текст |
| workflow | Список возможных значений: process\_1\_ideation; process\_2\_design; process\_3\_development; process\_4\_testing; process\_5\_deployment; process\_6\_production; process\_7\_stopped; process\_8\_rejected; | Текст |
| queue\_id | GUID из Оркестратора или номер очереди (любое уникальное значение, связанное с очередью в Оркестраторе). Может быть несколько значений, разделённых вертикальной чертой | Guid |
| effect\_planned | Планируемый эффект, в млн. рублей | Число |
| effect\_actual | Действительный эффект, в млн. рублей | Число |
| robot\_cost | Стоимость разработки робота, в млн. рублей | Число |
| project | Название проекта | Текст |
| period | Период оплаты | Текст. В формате QY, например Q22023 |
| financial\_position | Финансовая позиция | Текст |
| pfm\_customer | ПФМ Заказчика | Текст |
| pfm\_developer | ПФМ Исполнителя | Текст |
| budget\_program | Программа бюджета | Текст |
| robot\_type | Тип лицензии | Текст |
| robot\_user | Email пользователей разработчиков в IH, через запятую «,». | Email |
| machine\_name | Имя машины, как оно выгружается в файле выгрузки machines.csv в поле «name». Имена машин могут быть перечислены через запятую «,». | Текст |
| responsible\_analyst | Email пользователей разработчиков в IH, через запятую «,». | Email |
| responsible\_developer | Email пользователей разработчиков в IH, через запятую «,». | Email |
| stream\_coordinator | Email пользователей разработчиков в IH, через запятую «,». | Email |
| planned\_development\_time | Плановый срок разработки. В формате QY, например Q22023. | Текст |
| epic | Эпик | Текст |
| solution\_requirements | Требования к Решению. Ссылка (URL). | Ссылка |
| architecture\_solutions | Архитектура Решения. Ссылка (URL). | Ссылка |
| pmi | Программа и методика испытаний. Ссылка (URL). | Ссылка |
| tpsr | Тестовые планы и сценарии решения. Ссылка (URL). | Ссылка |
| test\_protocol | Протокол тестирования. Ссылка (URL). | Ссылка |
| psi\_protocol | Протокол ПСИ. Ссылка (URL). | Ссылка |
| tso | ТСО. Ссылка (URL). | Ссылка |
| effect\_calculation | Расчёт эффекта. Ссылка (URL) | |
| other | Ссылка на файл, который нужно загрузить и прикрепить к документу. Формат файла: «Название файла<вертикальная черта>URL файла,Название файла 2<вертикальная черта>URL файла 2» | |
| project\_orch | Идентификаторы проектов, связанные с этим процессом, перечисленные через запятую «,». | |
| contour | Контур, к которому относится процесс. | |
